<h2>🎮 The Most Comprehensive Free Fire API</h2>
<p>Welcome to the <strong>Free Fire API</strong> - a comprehensive, API service that provides access to Free Fire player statistics, craftland maps, wishlist data, and game assets.</p>
<h3>🌟 Key Features</h3>
<ul>
<li>✅ <strong>Real-time Player Statistics</strong> - Access live player data and rankings</li>
<li>✅ <strong>Comprehensive Profile Data</strong> - Get detailed player information</li>
<li>✅ <strong>Craftland Maps</strong> - Fetch custom map information</li>
<li>✅ <strong>Game Assets</strong> - Access in-game images and icons</li>
<li>✅ <strong>Ban Status Checking</strong> - Verify player ban status</li>
<li>✅ <strong>Multi-Region Support</strong> - SG, IND, BR regions</li>
<li>✅ <strong>Fast &amp; Reliable</strong> - Optimized for performance</li>
</ul>
<h2>🔑 Getting Your API Key</h2>
<p>To use this API, you&#x27;ll need an API key. Get yours at:
<strong><a href="https://developers.freefirecommunity.com/">Free Fire Community Developer Platform</a></strong></p>
<h3>Authentication Methods</h3>
<p>You can authenticate your API requests using either method:</p>
<ol>
<li>
<p><strong>Header Authentication (Recommended)</strong></p>
<pre><code>x-api-key: your_api_key_here
</code></pre>
<p>More secure as the key is not visible in URLs or logs.</p>
</li>
<li>
<p><strong>Query Parameter Authentication</strong></p>
<pre><code>?key=your_api_key_here
</code></pre>
<p>Simpler for testing but less secure. Use header method in production.</p>
</li>
</ol>
<h2>🌍 Regional Support</h2>
<p>Our API supports all Free Fire regions with three main endpoints:</p>
<ul>
<li><strong>SG Region</strong>: Covers <code>SG</code>, <code>ID</code>, <code>ME</code>, <code>VN</code>, <code>TH</code>, <code>CIS</code>, <code>EU</code>, <code>TW</code>, <code>MY</code>, <code>PK</code>, <code>BD</code></li>
<li><strong>IND Region</strong>: Covers India only</li>
<li><strong>BR Region</strong>: Covers <code>BR</code>, <code>US</code>, <code>NA</code>, <code>LATAM</code></li>
</ul>
<h2>🚀 Rate Limits</h2>
<p>Rate limits vary by plan:</p>
<ul>
<li><strong>Trial</strong>: Playground only — no API key, no production usage</li>
<li><strong>Trial Pass</strong>: 300 requests/24hr, 20 requests/minute, 100 requests/hour</li>
<li><strong>Beginner</strong>: 3,000 requests/month, 30 requests/minute, 500 requests/hour</li>
<li><strong>Developer</strong>: 12,000 requests/month, 60 requests/minute, 2,000 requests/hour</li>
<li><strong>Pro</strong>: 30,000 requests/month, 120 requests/minute, 6,000 requests/hour</li>
<li><strong>Enterprise</strong>: 500,000+ requests/month, custom rate limits</li>
</ul>
<p>Check response headers for your current rate limit status.</p>
